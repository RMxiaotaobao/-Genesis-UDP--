## 1. 软件简介

Genesis UDP 图传调试助手是一款专为智能车队设计的调试工具，支持：

- **实时变量监视** - 表格化显示下位机发送的变量数据
- **波形可视化** - 多种视图模式（单曲线、叠加图、仪表盘）
- **双向调参** - 实时修改下位机参数，无需重新烧录
- **MJPEG 图传** - 实时查看摄像头画面
- **数据记录** - CSV 格式保存调试数据

## 4. 快速上手

### 4.1 基本使用流程

```
1. 启动上位机程序
2. 配置 UDP 端口（默认 8080）
3. 下位机发送数据到 PC 的 IP:端口
4. 上位机自动解析并显示数据
```

### 4.2 第一次使用

**步骤 1：启动程序**

运行 `variable_monitor_v3.exe` 或 `python variable_monitor_v3.py`

**步骤 2：查看本机 IP**

程序启动后会显示本机 IP 地址，记住这个 IP。

**步骤 3：配置下位机**

在下位机代码中设置目标 IP 为你的本机 IP：

```cpp
const std::string TARGET_IP = "192.168.101.101";  // 替换为你的 PC IP
const uint16_t    TARGET_PORT = 8080;
```

**步骤 4：发送数据**

下位机发送格式：
```cpp
// 格式：name1:value1,name2:value2,name3:value3
std::string data =
"speed:12.3,
 error:-2.5,
 kp:1.0";
udp_client.udp_send_string(data);
```

**步骤 5：查看数据**

上位机会自动解析并显示在表格中。

## 5. 下位机配置

### 5.1 数据发送格式

**协议格式**：
```
name1:value1,name2:value2,name3:value3
```

- 逗号 `,` 分隔不同参数
- 冒号 `:` 分隔参数名和值
- 值为浮点数，推荐 `%.3f` 格式

**示例**：
```
speed_L:12.345,speed_R:11.234,err:-3.000,pos_out:45.678
```

### 5.2 代码示例

**最小示例**：

```cpp
#include "udp_tuner.hpp"

int main() {
    // 1. 创建调参器
    UdpTuner tuner("192.168.101.101", 8080);  // PC 的 IP 和端口
    if (!tuner.init()) {
        printf("UDP 初始化失败!\n");
        return -1;
    }

    // 2. 发送数据
    double speed = 0.0;
    while (true) {
        speed += 0.1;

        // 构造参数列表
        std::vector<ParamValue> params = {
            {"speed", speed},
            {"error", 0.0},
            {"kp", 1.0}
        };

        // 发送
        tuner.send_data(params);

        usleep(10000);  // 10ms 间隔
    }

    return 0;
}
```

**带接收功能的示例**：

```cpp
#include "udp_tuner.hpp"
#include <map>

int main() {
    UdpTuner tuner("192.168.101.101", 8080);
    if (!tuner.init()) {
        printf("UDP 初始化失败!\n");
        return -1;
    }

    double kp = 1.0, ki = 0.0, kd = 0.0;

    while (true) {
        // 发送数据
        std::vector<ParamValue> params = {
            {"kp", kp}, {"ki", ki}, {"kd", kd}
        };
        tuner.send_data(params);

        // 接收 PC 下发的参数
        std::map<std::string, double> cmd;
        if (tuner.receive_command(cmd)) {
            auto it = cmd.find("kp");
            if (it != cmd.end()) kp = it->second;

            it = cmd.find("ki");
            if (it != cmd.end()) ki = it->second;

            it = cmd.find("kd");
            if (it != cmd.end()) kd = it->second;
        }

        usleep(10000);  // 10ms
    }

    return 0;
}
```

### 5.3 图传配置

**HTTP MJPEG 图传**：

```cpp
#include "lq_net_image_trans.hpp"
#include "lq_camera_ex.hpp"

int main() {
    // 1. 启动 HTTP 服务器（端口 8080）
    lq_http_image_server server(8080);

    // 2. 初始化摄像头
    lq_camera_ex cam(160, 120, 120, LQ_CAMERA_0CPU_MJPG);

    // 3. 主循环
    while (true) {
        cv::Mat frame = cam.get_frame_raw();
        if (!frame.empty()) {
            server.push_frame(frame, 50);  // quality=50
        }
    }

    return 0;
}
```

**访问方式**：浏览器打开 `http://<板卡IP>:8080`

## 6. 功能详解

### 6.1 变量监视

**表格列说明**：

| 列名 | 说明 |
|------|------|
| 变量名 | 下位机发送的参数名 |
| 当前值 | 最新一次接收到的值 |
| 上次值 | 上一次接收到的值 |
| 变化量 | 当前值 - 上次值 |
| 最小值 | 历史最小值 |
| 最大值 | 历史最大值 |
| 平均值 | 历史平均值 |

**排序功能**：
- 点击列头可按该列排序
- 支持升序/降序切换

**过滤功能**：
- 支持按变量名过滤
- 支持按值范围过滤

### 6.2 波形显示

**视图模式**：

1. **单曲线视图**
   - 每个变量单独显示一条曲线
   - 适合观察单个变量的趋势

2. **叠加图视图**
   - 多个变量叠加在同一图表
   - 适合对比多个变量的关系

3. **仪表盘视图**
   - 数字仪表盘显示当前值
   - 适合监控关键参数

**操作说明**：
- 鼠标悬停查看具体数值
- 滚轮缩放时间轴
- 右键菜单切换视图模式

### 6.3 双向调参

**PC → 下位机**：

1. 在变量表格中找到要修改的变量
2. 双击"当前值"列
3. 输入新值并回车
4. 上位机自动发送到下位机

**批量调参**：
- 支持一次性修改多个参数
- 格式：`param1:100.0,param2:200.0`

### 6.4 MJPEG 图传

**配置方法**：

1. 下位机启动 HTTP 服务器
2. 在上位机配置图传地址：`http://<板卡IP>:8080/stream`
3. 点击"连接图传"按钮

**功能特性**：
- 支持多客户端同时查看
- 自动重连机制
- 帧率显示

### 6.5 数据记录

**录制功能**：
- 点击工具栏"录制"按钮
- 数据自动保存为 CSV 文件
- 支持自定义保存路径

**CSV 格式**：
```csv
timestamp,speed_L,speed_R,error,kp,ki,kd
1000,12.345,11.234,-3.000,1.000,0.100,0.050
1010,12.456,11.345,-2.900,1.000,0.100,0.050
```

## 7. 配置说明

### 7.1 配置文件

配置文件为 `config.json`，位于程序同级目录。

**常用配置项**：

```json
{
  "udp_port": 8080,
  "stream_url": "http://192.168.101.101:8080/stream",
  "show_splash": true,
  "dark_mode": false,
  "name_map": {
    "speed_L": "左轮速度",
    "speed_R": "右轮速度"
  },
  "value_map": {
    "state": {
      "0": "停止",
      "1": "运行",
      "2": "调试"
    }
  },
  "var_types": {
    "speed_L": "float2",
    "state": "enum"
  }
}
```

### 7.2 配置项说明

| 配置项 | 类型 | 说明 |
|--------|------|------|
| `udp_port` | int | UDP 监听端口，默认 8080 |
| `stream_url` | string | MJPEG 图传地址 |
| `show_splash` | bool | 是否显示启动画面 |
| `dark_mode` | bool | 是否启用深色模式 |
| `name_map` | object | 变量名重映射（原始名 → 显示名） |
| `value_map` | object | 枚举值映射 |
| `var_types` | object | 变量显示类型（int/float1/float2/float3/enum） |

### 7.3 变量类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `int` | 整数显示 | 123 |
| `float1` | 1位小数 | 12.3 |
| `float2` | 2位小数 | 12.34 |
| `float3` | 3位小数 | 12.345 |
| `enum` | 枚举显示 | 运行/停止 |

## 8. 常见问题

### Q1: 上位机收不到数据？

**检查清单**：

1. ✅ 网络连通性
   ```bash
   ping <板卡IP>
   ```

2. ✅ IP 地址正确
   - `TARGET_IP` 必须是 **PC 的 IP**，不是板卡的 IP

3. ✅ 端口一致
   - 上位机监听端口 = 下位机发送端口

4. ✅ 防火墙设置
   - Windows：允许 Python/程序通过防火墙
   - Linux：检查 `ufw` 或 `iptables`

5. ✅ 数据格式正确
   - 格式：`name:value,name:value`
   - 编码：UTF-8

### Q2: 波形显示卡顿？

**解决方案**：

1. 降低发送频率（从 100Hz 降到 50Hz）
2. 减少变量数量
3. 关闭不需要的视图模式

### Q3: 图传画面延迟高？

**优化方法**：

1. 降低 JPEG 质量（从 50 降到 30）
2. 降低分辨率（从 320x240 改为 160x120）
3. 使用 UDP 直传替代 HTTP（延迟更低）

### Q4: 如何修改变量显示名称？

编辑 `config.json` 的 `name_map`：

```json
{
  "name_map": {
    "speed_L": "左轮速度",
    "speed_R": "右轮速度",
    "err": "误差",
    "state": "状态"
  }
}
```

### Q5: 如何导出调试数据？

1. 点击工具栏"录制"按钮
2. 调试完成后点击"停止"
3. CSV 文件自动保存到程序目录
